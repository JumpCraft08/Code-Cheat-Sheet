C++ es un lenguaje compilado y sensible a mayúsculas. El punto de entrada es siempre la función main. Puntos clave:

  • Uso de directivas de preprocesador como #include.
  • El espacio de nombres std contiene las funciones estándar.
  • Cada instrucción termina con punto y coma (;).
  1. #include <iostream>
  3. int main() {
  4. std::cout << "Hola Mundo" << std::endl;
  5. return 0;
  6. }

En proyectos de C++, es común separar la declaración de la implementación:

Extensión Descripción
.cpp Archivos de código fuente (implementación).
.h / .hpp Headers (cabeceras): declaraciones y prototipos.
.exe / .out Archivo ejecutable después de la compilación.

Compilación básica con GCC: g++ main.cpp -o programa

Para interactuar con la consola se utiliza la librería <iostream>:

  • cout: Flujo de salida (imprimir).
  • cin: Flujo de entrada (leer datos).
  • endl: Inserta un salto de línea y limpia el búfer.
  1. int edad;
  2. std::cout << "Edad: ";
  3. std::cin >> edad; // Lee del teclado

C++ tiene varios tipos de datos integrados para representar diferentes valores:

Se pueden añadir modificadores para alterar el rango o precisión de los tipos:

  • unsigned: Solo valores positivos.
  • long / short: Aumenta o disminuye el tamaño en memoria.
  1. unsigned int p = 500;
  2. long double precision = 0.00001;
  3. short int pequeño = 10;

Una variable es un espacio de memoria con un nombre. En C++ es obligatorio declarar el tipo.

  1. int edad = 25; // Inicialización
  2. const float PI = 3.1416; // Constante
  3. auto variable = 10; // Tipo deducido (C++11)

El lugar donde declaras una variable define su vida útil:

  • Locales: Dentro de una función o bloque { }.
  • Globales: Fuera de todas las funciones.
  1. int global = 100;
  2. void func() {
  3. int local = 5;
  4. }

Cadenas al estilo C: arreglos de caracteres terminados en nulo (\0).

Determinan el flujo de ejecución de un programa. Sin ellas, el código se ejecutaría de forma estrictamente lineal de arriba hacia abajo.

  • Selección: Elegir entre diferentes caminos (if, switch).
  • Iteración: Repetir bloques de código (bucles for, while).
  • Salto: Transferir el control a otra parte del programa.

C++ utiliza bloques delimitados por { } para agrupar las instrucciones que pertenecen a una estructura.

C++ evalúa expresiones para decidir el flujo. Cualquier valor entero distinto de cero es tratado como true.

  1. bool condicion = true;
  3. if (condicion) {
  4. // Se ejecuta si la condición es verdadera
  5. }

La correcta indentación y el uso de llaves son fundamentales para evitar errores lógicos en estructuras anidadas.

Evalúa una expresión booleana para decidir qué bloque ejecutar.

  1. if (puntos >= 90) {
  2. cout << "Excelente";
  3. } else if (puntos >= 60) {
  4. cout << "Aprobado";
  5. } else {
  6. cout << "Reprobado";
  7. }

Ideal para múltiples condiciones basadas en un mismo valor integral (int, char).

  1. switch (opcion) {
  2. case 1: iniciar(); break;
  3. case 2: salir(); break;
  4. default: error();
  5. }

*No olvides el break para evitar el "fall-through".

Se usa cuando se conoce de antemano el número de iteraciones.

  1. // (inicialización; condición; actualización)
  2. for (int i = 0; i < 10; i++) {
  3. cout << i << " ";
  4. }

También existe el range-based for (C++11):
for (int x : miArreglo) { ... }

Dependen de una condición que se evalúa en cada vuelta.

  • while: Evalúa antes de entrar.
  • do-while: Evalúa después (se ejecuta al menos una vez).
  1. while (condicion) { /* ... */ }
  2. do {
  3. // Código ejecutado una vez mínimo
  4. } while (condicion);

Controlan el flujo dentro de los bucles:

  • break: Sale completamente del bucle actual.
  • continue: Salta el resto del código y va a la siguiente iteración.
  1. if (error) break;
  2. if (omitir) continue;
  • return: Finaliza la ejecución de una función y devuelve un valor.
  • goto: Transfiere el control a una etiqueta (no recomendado en programación moderna).
  1. int suma() { return a + b; }
  2. goto etiqueta; // ¡Usar con precaución!

En C++, los errores se categorizan según el momento en que se detectan:

  • Compilación: Errores de sintaxis o tipos detectados por el compilador.
  • Enlace (Linker): Referencias a funciones o variables no definidas.
  • Tiempo de ejecución: El programa falla mientras corre (división por cero, acceso inválido).
  • Lógicos: El programa corre pero produce resultados incorrectos.

Una excepción es una señal que indica un problema inusual durante la ejecución. C++ permite separar el código de detección de errores del código de manejo.

La librería estándar provee la jerarquía <stdexcept>:

  • std::runtime_error
  • std::logic_error
  • std::out_of_range

Se utiliza para verificar supuestos durante el desarrollo. Requiere <cassert>.

  1. // Si la condición es falsa, el programa aborta
  2. assert(puntero != nullptr);
  3. assert(divisor != 0);

Nota: Los assert se eliminan automáticamente si se compila en modo "Release" (con NDEBUG).

Se utiliza try para envolver código propenso a errores y catch para manejarlos:

  1. try {
  2. if (y == 0) throw "Div por cero";
  3. res = x / y;
  4. } catch (const char* msg) {
  5. cerr << msg << endl;
  6. }

Es mejor práctica lanzar y atrapar objetos de excepción por referencia:

  1. try {
  2. vector.at(999);
  3. } catch (const std::exception& e) {
  4. cout << e.what();
  5. }

La OOP es un paradigma basado en "objetos" que contienen datos y funciones. Una clase es el molde, y el objeto es la instancia real.

  1. class Coche {
  2. public:
  3. string marca;
  4. void encender() { /*...*/ }
  5. };
  7. Coche miAuto; // Instanciación

Es un método especial que se ejecuta automáticamente al crear un objeto. Tiene el mismo nombre que la clase y no retorna nada.

  1. class Robot {
  2. public:
  3. Robot(string n) { nombre = n; }
  4. private:
  5. string nombre;
  6. };

Es la forma más eficiente de inicializar miembros en C++:

  1. Robot(string n) : nombre(n) {
  2. // Cuerpo del constructor
  3. }

Son funciones definidas dentro de la clase. Pueden ser definidos dentro o fuera (usando el operador ::).

  1. void Coche::acelerar() {
  2. velocidad += 10;
  3. }

El puntero this representa al objeto actual que invoca el método.

Controlan qué partes del código pueden ver los miembros de la clase:

  • public: Accesible desde cualquier lugar.
  • private: Solo accesible dentro de la misma clase.
  • protected: Accesible en la clase y sus herederos.

Por defecto, en una class todo es private.

Consiste en mostrar solo las funciones esenciales ocultando los detalles de implementación internos.

  1. class DB {
  2. public:
  3. void conectar(); // Interfaz pública
  4. private:
  5. void abrirSocket(); // Detalles ocultos
  6. };

Es la agrupación de datos y métodos en una unidad, restringiendo el acceso directo a los detalles internos mediante métodos públicos (Getters y Setters).

  1. class Cuenta {
  2. private: double saldo;
  3. public:
  4. void setSaldo(double s) { if(s > 0) saldo = s; }
  5. double getSaldo() { return saldo; }
  6. };

Permite que una clase (hija) derive atributos y métodos de otra (padre), facilitando la reutilización de código.

  1. class Animal {
  2. public: void comer();
  3. };
  5. class Perro : public Animal {
  6. public: void ladrar();
  7. };

Ocurre cuando hay una jerarquía de clases relacionadas por herencia. Permite invocar métodos de la clase derivada a través de punteros de la clase base.

  1. virtual void hacerSonido() {
  2. cout << "Sonido genérico";
  3. }

Se usa la palabra clave virtual en la base y override en la derivada.

Tipos definidos por el usuario que consisten en un conjunto de constantes con nombre.

  1. enum Estado { APAGADO, ENCENDIDO, STANDBY };
  3. enum class Color { Rojo, Azul }; // Strong typed
  4. Color c = Color::Rojo;

Permiten escribir código independiente del tipo de dato. El compilador genera la versión necesaria automáticamente.

  1. template <typename T>
  2. T maximo(T a, T b) {
  3. return (a > b) ? a : b;
  4. }

Una clase declarada dentro de otra. Se usa para organizar clases que solo tienen sentido dentro del contexto de la clase contenedora.

  1. class Ordenador {
  2. class Procesador {
  3. // Solo relevante para Ordenador
  4. };
  5. };

C++ permite un control granular de la memoria y el comportamiento de las clases mediante:

  • Herencia Múltiple: Una clase puede derivar de varias bases.
  • Interfaces (Clases Abstractas): Clases con al menos un método virtual puro.
  • RTTI: Identificación de tipos en tiempo de ejecución.
  1. class Interfaz {
  2. virtual void metodo() = 0; // Virtual puro
  3. };

Son funciones anónimas que pueden capturar variables de su entorno. Introducidas en C++11.

  1. // [captura](parámetros) -> retorno { cuerpo }
  2. auto suma = [](int a, int b) {
  3. return a + b;
  4. };
  5. int res = suma(5, 3);

Útiles para algoritmos de la STL como std::sort o std::for_each.

Diferencias clave en el acceso a memoria:

  • Puntero: Almacena una dirección. Puede ser nulo o reasignado.
  • Referencia: Alias de una variable existente. No puede ser nula ni reasignada.
  1. int x = 10;
  2. int* ptr = &x; // Puntero
  3. int& ref = x; // Referencia
  4. *ptr = 20; // Acceso por desreferencia

Técnica donde la gestión de recursos (memoria, archivos, mutex) está ligada al ciclo de vida de un objeto.

  • El recurso se adquiere en el constructor.
  • El recurso se libera automáticamente en el destructor.

Es la base de los Smart Pointers (unique_ptr, shared_ptr).

Permite "transferir" recursos de un objeto a otro en lugar de copiarlos, mejorando drásticamente el rendimiento.

  1. string a = "Largo...";
  2. string b = std::move(a);

Utiliza Rvalue References (&&) para identificar objetos temporales que pueden ser "vaciados".

La concurrencia permite que un programa realice múltiples tareas "al mismo tiempo". C++ proporciona soporte nativo desde C++11 mediante la librería <thread>.

  • Concurrencia: Varias tareas progresan simultáneamente (intercaladas).
  • Paralelismo: Varias tareas se ejecutan físicamente al mismo tiempo en diferentes núcleos.

Un std::thread representa un hilo de ejecución individual.

  1. void tarea(int id) { /*...*/ }
  3. std::thread t1(tarea, 1);
  4. t1.join(); // Espera a que termine

Es crucial llamar a join() o detach() antes de que el objeto thread sea destruido para evitar que el programa aborte.

std::async ejecuta una función (potencialmente en un hilo separado) y devuelve un std::future que contendrá el resultado.

  1. auto fut = std::async(calcular);
  2. // ... realizar otras tareas ...
  3. int res = fut.get(); // Bloquea hasta tener el valor

Para evitar "Race Conditions", se usan Mutex para proteger datos compartidos.

  1. std::mutex mtx;
  2. void seguro() {
  3. std::lock_guard<std::mutex> lock(mtx);
  4. // El mutex se libera solo al salir del scope (RAII)
  5. contador++;
  6. }

Permiten que un hilo espere a que otro hilo cumpla una condición específica.

  1. std::condition_variable cv;
  2. // Hilo 1: espera
  3. cv.wait(lock, []{ return listo; });
  4. // Hilo 2: notifica
  5. listo = true;
  6. cv.notify_one();

Desde C++17, muchos algoritmos de la STL pueden ejecutarse en paralelo pasando una política de ejecución.

  1. #include <execution>
  3. std::sort(std::execution::par, v.begin(), v.end());

Políticas comunes:

  • seq: Secuencial (por defecto).
  • par: Paralelo.
  • par_unseq: Paralelo y vectorizado.

Una estructura de datos es una forma particular de organizar información en un computador para que pueda ser utilizada de manera eficiente.

  • Lineales: Los elementos siguen una secuencia (Arrays, Listas, Pilas).
  • No Lineales: Relaciones jerárquicas o complejas (Árboles, Grafos).

La elección depende de la operación que necesites optimizar: búsqueda, inserción o borrado.

Colección de elementos del mismo tipo almacenados en posiciones contiguas de memoria. Su tamaño se define al compilar.

  1. int numeros[5] = {10, 20, 30};
  2. int valor = numeros[0]; // Acceso por índice

Ventaja: Acceso instantáneo por índice $O(1)$.
Desventaja: No puede cambiar su tamaño.

Es un contenedor moderno que envuelve a los arreglos tradicionales, añadiendo seguridad y funciones miembro.

  1. #include <array>
  2. std::array<int, 3> miArray = {1, 2, 3};
  3. int tam = miArray.size();

A diferencia de los arreglos clásicos, std::array no decae a puntero automáticamente y conoce su propio tamaño.

Es un arreglo dinámico que puede cambiar de tamaño automáticamente. Los elementos se almacenan de forma contigua.

  1. #include <vector>
  2. std::vector<int> v = {1, 2, 3};
  3. v.push_back(4); // Agrega al final
  4. int n = v[0]; // Acceso O(1)

Es el contenedor por defecto más recomendado en C++ por su eficiencia con el caché.

Contenedor que almacena elementos únicos siguiendo un orden específico (por defecto ascendente).

  1. #include <set>
  2. std::set<int> s;
  3. s.insert(10);
  4. s.insert(10); // No se inserta (ya existe)

Implementado usualmente como un árbol binario equilibrado. Búsqueda en $O(\log n)$.

Almacena pares clave-valor. Las claves son únicas y están ordenadas.

  1. #include <map>
  2. std::map<string, int> edades;
  3. edades["Ana"] = 25;
  4. int e = edades["Ana"];

Para acceso más rápido sin orden, existe unordered_map (basado en Hash).

  • Queue: Estructura FIFO (First-In, First-Out).
  • Deque: Permite inserción/extracción eficiente en ambos extremos.
  1. std::queue<int> q;
  2. q.push(1); // Encola
  3. q.pop(); // Desencola

Estructura LIFO (Last-In, First-Out). Solo se tiene acceso al elemento superior.

  1. std::stack<int> s;
  2. s.push(100); // Apila
  3. int tope = s.top();
  4. s.pop(); // Desapila

Permiten definir el orden en sets, maps o algoritmos de ordenamiento.

  1. struct Comp {
  2. bool operator()(int a, int b) const {
  3. return a > b; // Orden descendente
  4. }
  5. };

Objetos que actúan como punteros para recorrer los elementos de un contenedor.

  1. auto it = v.begin();
  2. while (it != v.end()) {
  3. cout << *it;
  4. ++it;
  5. }

Funciones genéricas que operan sobre rangos de elementos.

  1. sort(v.begin(), v.end());
  2. find(v.begin(), v.end(), 5);
  3. reverse(v.begin(), v.end());

Proporcionan soluciones eficientes para búsqueda, ordenamiento y manipulación.

C++ no tiene una librería de networking nativa en el estándar hasta la fecha (aunque existe la propuesta Networking TS). Se suelen usar librerías de terceros o APIs del sistema operativo:

  • Boost.Asio: El estándar de facto para red y E/S de bajo nivel.
  • POCO: Un framework completo para aplicaciones en red.
  • WinSock / POSIX Sockets: APIs nativas de Windows y Unix.

Un socket es un punto final de una comunicación bidireccional entre dos programas en la red.

  1. // Flujo básico en servidor (Unix/Win)
  2. socket(); // Crear
  3. bind(); // Asignar IP/Puerto
  4. listen(); // Esperar conexión
  5. accept(); // Aceptar cliente

Los sockets pueden ser Stream (TCP, orientado a conexión) o Datagram (UDP, sin conexión).

Protocolo de nivel de aplicación para sistemas de información distribuidos. En C++ se suelen usar librerías como libcurl o httplib.

  1. // Ejemplo conceptual con httplib
  2. httplib::Client cli("http://api.com");
  3. auto res = cli.Get("/data");
  4. if (res && res->status == 200) {

C++ otorga control total sobre la memoria, lo que introduce riesgos si no se programa con cuidado. Los pilares de la seguridad son:

  • Memoria Segura: Evitar desbordamientos de búfer (Buffer Overflow).
  • Validación: Nunca confiar en la entrada del usuario (Sanitización).
  • Principio de Privilegio Mínimo: Exponer solo lo estrictamente necesario.

Se recomienda usar contenedores estándar (como std::vector) en lugar de arreglos de C para mitigar vulnerabilidades.

El cifrado transforma la información para que solo personas autorizadas puedan leerla. Se divide en dos tipos:

  • Simétrico: Misma clave para cifrar y descifrar (ej. AES).
  • Asimétrico: Clave pública para cifrar y privada para descifrar (ej. RSA).

En C++, librerías como OpenSSL o Crypto++ son el estándar para implementar estos algoritmos de forma segura.

Transforma el código fuente (legible por humanos) en código máquina (ejecutable). Consta de cuatro etapas principales:

  1. Preprocesado: Manejo de directivas (#include, #define).
  2. Compilación: Traducción a código ensamblador.
  3. Ensamblado: Creación de archivos objeto (código binario).
  4. Enlazado (Linking): Unión de objetos y librerías en un ejecutable final.

  3. // -O2: Optimización de rendimiento
  • Make: Utiliza un Makefile para automatizar la compilación mediante reglas de dependencia.
  • CMake: Un generador de sistemas de construcción meta-plataforma.

C++ utiliza el concepto de streams (flujos) para manejar la comunicación entre el programa y los dispositivos externos.

  • cin: Flujo de entrada estándar (teclado).
  • cout: Flujo de salida estándar (consola).
  • cerr: Flujo de error (salida sin buffer para errores críticos).

El operador << se usa para insertar datos en el flujo y >> para extraerlos.

Introducida en C++17, permite manipular rutas, directorios y metadatos de archivos de forma multiplataforma.

  1. namespace fs = std::filesystem;
  2. if (fs::exists("config.txt")) {
  3. auto tam = fs::file_size("config.txt");
  4. }

Es fundamental para tareas como crear carpetas, borrar archivos o listar el contenido de un directorio.

La cabecera <fstream> define clases para operar con archivos en el disco:

  • ifstream: Para lectura (Input).
  • ofstream: Para escritura (Output).
  • fstream: Para ambas operaciones.
  1. ofstream archivo("datos.txt");
  2. archivo << "Guardando texto..." << endl;
  3. archivo.close();

std::stringstream permite tratar una cadena de texto como si fuera un flujo, facilitando la conversión de tipos.

  1. stringstream ss;
  2. ss << 100 << " " << 45.5;
  3. int i; double d;
  4. ss >> i >> d; // Extrae valores formateados

Es muy útil para parsear líneas de texto complejas o construir mensajes dinámicos.

Proceso de convertir un objeto en memoria a un formato que pueda ser almacenado o transmitido (binario, JSON, XML).

  • Binaria: Rápida pero difícil de leer por humanos.
  • Texto (JSON/XML): Legible y compatible con otros lenguajes.

En C++ moderno se suelen usar librerías como Cereal o nlohmann/json para automatizar este proceso.

C++ ofrece un control directo sobre la memoria del sistema. El programador decide cuándo y dónde se asignan los recursos.

  • Estática: Variables globales o static. Existen durante toda la vida del programa.
  • Automática: Variables locales en funciones (Stack).
  • Dinámica: Solicitada explícitamente en tiempo de ejecución (Heap).

Un manejo ineficiente puede degradar el rendimiento o causar cierres inesperados (Crashes).

Característica Stack (Pila) Heap (Montículo)
Gestión Automática Manual (new/delete)
Velocidad Muy rápida Más lenta
Tamaño Limitado (fijo) Flexible (grande)

El Stack se gestiona por el sistema al entrar/salir de scopes. El Heap permite que los datos sobrevivan a la función que los creó.

Desde C++11, se recomienda evitar punteros crudos y usar Smart Pointers para gestionar el ciclo de vida automáticamente (RAII).

  • std::unique_ptr: El objeto tiene un único dueño. No se puede copiar.
  • std::shared_ptr: El objeto tiene múltiples dueños; se libera cuando el último desaparece.
  1. auto p = std::make_unique<MiClase>();
  2. // Se libera automáticamente al final del scope

Ocurre cuando se reserva memoria en el Heap (new) y se pierde el puntero sin haber liberado el espacio (delete).

  1. void fuga() {
  2. int* p = new int(10);
  3. return; // ¡Error! Puntero perdido, memoria ocupada
  4. }

Herramientas como Valgrind o el uso de Smart Pointers son las mejores defensas para prevenir estos fallos.

C++ incluye características versátiles que no encajan en una sola categoría pero son esenciales para el desarrollo profesional:

  • Type Aliasing: Uso de using o typedef para crear nombres legibles.
  • Atributos: Anotaciones como [[nodiscard]] o [[maybe_unused]] para el compilador.
  • Llamadas al Sistema: Interacción directa con el SO mediante librerías nativas.

El preprocesador actúa antes de la compilación real, procesando líneas que comienzan con #.

  1. #define MAX_BUFFER 1024 // Macro de reemplazo
  2. #ifdef VERSION_PRO // Compilación condicional
  3. activarFunciones();
  4. #endif

Nota: En C++ moderno se prefiere constexpr y inline sobre las macros tradicionales por seguridad de tipos.

C++ es un lenguaje vivo regulado por ISO. Cada versión añade potencia y simplifica la sintaxis:

  • C++11: La gran revolución (Smart pointers, lambdas, auto).
  • C++17: Filesystem, variantes, std::optional.
  • C++20: Módulos, Conceptos y Corrutinas.
  • C++23: Mejoras en la STL y soporte para std::print.

El desarrollo profesional en C++ va más allá del código; implica un entorno robusto de herramientas:

  • Gestores de Paquetes: Conan y vcpkg para manejar dependencias.
  • Testing: GoogleTest (gTest) y Catch2.
  • Análisis Estático: Clang-Tidy y Cppcheck para encontrar errores sin ejecutar.
  • IDEs: Visual Studio, CLion y VS Code con extensiones de C++.
