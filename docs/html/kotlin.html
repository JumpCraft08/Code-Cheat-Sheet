Para permitir que una variable contenga el valor null, debemos declarar el tipo con un signo de interrogación ?:

  1. var b: String? = "Hola"
  2. b = null // Correcto
  4. val l = b.length // Error: b podría ser nulo

Si intentas acceder a una propiedad de un tipo nullable directamente, Kotlin te obligará a usar un operador de seguridad.

El operador ?. ejecuta la llamada solo si la referencia no es nula. Si es nula, devuelve null en lugar de lanzar una excepción:

  1. val nombre: String? = null
  3. println(nombre?.length)
  4. // Imprime "null" sin romper el programa

También es muy útil para cadenas de llamadas (encadenamiento):

  1. // Si alguno es null, el resultado es null
  2. user?.address?.city?.name

El operador Elvis ?: permite definir un valor alternativo en caso de que la expresión de la izquierda sea null:

  1. val b: String? = null
  3. val l = b?.length ?: 0
  4. // Si b es null, l vale 0

Incluso puedes usarlo para lanzar excepciones o retornar funciones:

  1. val s = str ?: return false
  2. val d = data ?: error("Sin datos")

Kotlin ofrece herramientas potentes que van más allá de la POO tradicional:

Estas características permiten escribir código más expresivo, seguro y fácil de mantener, reduciendo el "boilerplate" típico de otros lenguajes.

Las lambdas son funciones anónimas que se pueden tratar como valores. Se encierran entre llaves { }:

  1. // Sintaxis: { params -> cuerpo }
  2. val sumar = { a: Int, b: Int -> a + b }
  4. // Si solo hay un parámetro, se usa 'it'
  5. val doble = { it * 2 }
  7. println(sumar(5, 10)) // 15

Son funciones que reciben otras funciones como parámetros o devuelven una función:

  1. fun operar(a: Int, b: Int, operacion: (Int, Int) -> Int) {
  2. val res = operacion(a, b)
  3. println("Resultado: $res")
  4. }
  6. // Invocación
  7. operar(5, 5) { x, y -> x * y }

Si la función es el último parámetro, se puede sacar fuera de los paréntesis (Trailing Lambda).

Las sealed class representan jerarquías de clases restringidas. Son como Enums pero con estado propio por instancia:

Al usarlas en un when, el compilador verifica que cubras todos los casos (Exhaustividad).

Las corrutinas son "hilos ligeros". Permiten ejecutar código asíncrono de forma secuencial sin bloquear el hilo principal (UI Thread):

  • Suspensión: Una corrutina puede pausarse sin bloquear el hilo donde se ejecuta.
  • Ligereza: Puedes ejecutar miles de corrutinas con un consumo de memoria mínimo comparado con los Threads.
  1. // No bloquea, solo suspende
  2. delay(1000)

Existen dos formas principales de iniciar una corrutina dependiendo de si esperamos un resultado o no:

  • launch: "Dispara y olvida". Devuelve un Job.
  • async: Devuelve un Deferred<T>. Se usa para obtener un valor con .await().
  1. val job = launch { hacerTarea() }
  3. val deferred = async { obtenerDato() }
  4. val resultado = deferred.await()

Los Dispatchers indican en qué hilo o grupo de hilos se ejecutará la corrutina:

Dispatcher Uso recomendado
MainUI, interacciones con el usuario.
IORed, bases de datos, archivos.
DefaultUso intensivo de CPU (filtros, cálculos).
UnconfinedNo confinado a un hilo específico.
  1. launch(Dispatchers.IO) {
  2. // Código de red aquí
  3. }

El CoroutineScope controla el ciclo de vida de las corrutinas. Si el scope se cancela, todas sus corrutinas mueren:

  • GlobalScope: Vive mientras dure la app (no recomendado).
  • viewModelScope: (Android) Muere con el ViewModel.
  • lifecycleScope: (Android) Muere con la Activity/Fragment.
  1. class MyActivity : Scope {
  2. fun load() {
  3. lifecycleScope.launch { ... }
  4. }
  5. }

Un Flow emite múltiples valores de forma secuencial a lo largo del tiempo (similar a una lista, pero asíncrona):

  4. emit(i) // Emite valor
  5. }
  6. }
  8. // Para consumirlo:
  9. numFlow().collect { v -> print(v) }

  • Solo pueden ser llamadas desde otra suspend function o desde una corrutina.

  3. return "Dato: $res"

Kotlin proporciona una jerarquía clara para manejar grupos de objetos. Se divide principalmente en tres tipos:

  • List: Colección ordenada con acceso por índice. Admite duplicados.
  • Set: Colección de elementos únicos. No garantiza orden.
  • Map: Conjunto de pares clave-valor. Las claves son únicas.

A diferencia de Java, Kotlin separa las interfaces en Lectura y Mutables.

Los Array tienen un tamaño fijo. Se representan por la clase Array o clases especializadas para tipos primitivos:

  1. // Array de Strings
  2. val arr = arrayOf("A", "B", "C")
  4. // Arrays de tipos primitivos (más eficientes)
  5. val ints = intArrayOf(1, 2, 3)
  7. // Acceso
  8. println(arr[0]) // "A"

Es la colección más común. Permite elementos repetidos y mantiene el orden de inserción:

  1. val lista = listOf("Kotlin", "Java", "C#")
  3. println(lista.first()) // "Kotlin"
  4. println(lista[1]) // "Java"
  5. println(lista.last()) // "C#"

Nota: listOf crea una lista inmutable (solo lectura).

Un Set no permite elementos duplicados. Útil para verificar pertenencia:

  1. val nums = setOf(1, 2, 3, 1)
  3. println(nums.size) // 3 (el duplicado se ignora)
  4. println(nums.contains(2)) // true

Los mapas almacenan pares. Las claves son únicas, pero los valores pueden repetirse:

  1. val capitales = mapOf(
  2. "ES" to "Madrid",
  3. "FR" to "París"
  4. )
  6. println(capitales["ES"]) // "Madrid"
  7. println(capitales.keys) // ["ES", "FR"]

Un Iterator permite recorrer una colección secuencialmente sin exponer su estructura interna. Es la base de los bucles for:

  • hasNext(): Verifica si quedan elementos.

  5. println(it.next())

  5. if (it.next() % 2 == 0) {

  7. }

Kotlin ofrece funciones de orden superior para procesar colecciones de forma declarativa:

  • map: Transforma cada elemento.
  • filter: Selecciona elementos bajo una condición.
  • any / all: Comprueba condiciones lógicas.
  1. val numbers = listOf(1, 2, 3, 4)
  3. val pares = numbers.filter { it % 2 == 0 }
  4. val dobles = numbers.map { it * 2 }

Operaciones para reducir la colección a un único valor o encontrar elementos específicos:

  • reduce / fold: Acumula valores.
  • find: Devuelve el primero que cumpla la condición (o null).
  • groupBy: Crea un mapa agrupando por una clave.
  1. // Suma todos los elementos
  2. val suma = numbers.reduce { acc, n -> acc + n }
  4. // Agrupar por paridad
  5. val grupos = numbers.groupBy {

  7. }

Kotlin aprovecha la infraestructura de la JVM para ofrecer seguridad robusta. Los pilares básicos incluyen:

  • Inmutabilidad: El uso de val previene modificaciones accidentales de datos sensibles.
  • Tipado Seguro: Evita desbordamientos de memoria y errores comunes en tiempo de ejecución.
  • Manejo de Nulos: Previene vulnerabilidades causadas por referencias nulas inesperadas.

Es fundamental no guardar contraseñas en texto plano y utilizar siempre TLS/HTTPS para la transmisión de datos.

El cifrado permite proteger datos mediante una clave. AES es el estándar recomendado para cifrado simétrico:

  1. val cipher = Cipher.getInstance("AES/GCM/NoPadding")
  2. val keySpec = SecretKeySpec(keyBytes, "AES")
  4. // Encriptar
  5. cipher.init(Cipher.ENCRYPT_MODE, keySpec)
  6. val encrypted = cipher.doFinal(textBytes)

Nota: Utiliza siempre modos seguros como GCM en lugar de ECB para evitar patrones en los datos cifrados.

El hashing crea una "huella digital" de un dato. Para contraseñas, se debe usar algoritmos como BCrypt o Argon2, nunca MD5 o SHA1:

  1. // Ejemplo básico con SHA-256 (solo para integridad)
  2. fun hash(input: String): String {
  3. val bytes = MessageDigest.getInstance("SHA-256")
  4. .digest(input.toByteArray())
  5. return bytes.joinToString("") { "%02x".format(it) }
  6. }

Salting: Añadir un valor aleatorio único (salt) antes del hashing es vital para prevenir ataques de tablas arcoíris.

La Stdlib proporciona las herramientas esenciales para el día a día sin necesidad de dependencias externas. Incluye:

  • Tipos intrínsecos: Manejo de rangos, progresiones y pares (Pair, Triple).

La clase Regex facilita la búsqueda y validación de patrones en cadenas de texto:

  1. val patron = """\d+""".toRegex()

A diferencia de lenguajes puramente interpretados, Kotlin pasa por un proceso de compilación que transforma el código fuente (.kt) en un formato ejecutable:

  • Análisis: El compilador verifica la sintaxis y los tipos de datos.
  • Generación: Se produce el código intermedio o final según el destino.
  • Optimización: Se eliminan redundancias y se aplican funciones inline.

En el ecosistema JVM, el resultado son archivos .class con bytecode compatible con Java.

El compilador de línea de comandos se llama kotlinc. Permite compilar archivos individuales o proyectos completos:

  1. // Compilar a Bytecode de Java
  2. kotlinc Main.kt -include-runtime -d Main.jar
  4. // Ejecutar el archivo generado
  5. java -jar Main.jar

El flag -include-runtime es necesario para que el JAR sea autónomo e incluya las librerías estándar de Kotlin.

Kotlin es un lenguaje Multiplataforma. Dependiendo del backend del compilador, puede ejecutarse en distintos entornos:

Destino Salida Uso Común
Kotlin/JVM Bytecode (JAR) Android, Backend (Spring/Ktor).
Kotlin/JS JavaScript Web Frontend, React, Node.js.
Kotlin/Native Binario (EXE/Mach-O) iOS, Windows, Linux, Embedded.

KDoc es el lenguaje de documentación de Kotlin (similar a Javadoc). Utiliza Markdown para el formato:

  1. /**
  2. * Calcula el área de un círculo.
  3. *
  4. * @param radius El radio en metros.
  5. * @return El área calculada.
  6. */
  7. fun calculateArea(radius: Double): Double { ... }

KDoc soporta etiquetas especiales para estructurar la información técnica:

  • @param: Describe un parámetro de función.
  • @return: Describe el valor de retorno.
  • @throws / @exception: Indica qué excepciones puede lanzar.

Para generar sitios web de documentación a partir de estos comentarios, se utiliza la herramienta Dokka.

Kotlin utiliza las librerías de la JVM para el manejo de I/O, pero añade funciones de extensión que simplifican enormemente las tareas comunes:

  • Salida estándar: print() y println().
  • Entrada estándar: readln() (disponible desde Kotlin 1.6).
  • Interoperabilidad: Acceso total a java.io y java.nio.

La ventaja de Kotlin es que reduce el "boilerplate" necesario para abrir y cerrar flujos de datos.

Kotlin permite leer el contenido de un archivo en una sola línea de código de forma segura:

  1. import java.io.File
  3. val file = File("datos.txt")
  5. // Leer todo el texto
  6. val contenido = file.readText()
  8. // Leer líneas como lista
  9. val lineas = file.readLines()

Escribir datos también es directo, gestionando automáticamente la apertura y cierre del archivo:

  1. val file = File("output.txt")
  3. // Sobrescribir contenido
  4. file.writeText("Hola Kotlin")
  6. // Añadir al final (Append)
  7. file.appendText("\nNueva línea")

Para archivos grandes, los Streams procesan los datos por partes (buffers) sin cargar todo en memoria:

  1. val file = File("grande.log")
  3. // Uso de 'use' para cerrar el stream automáticamente
  4. file.inputStream().bufferedReader().use { reader ->
  5. println(reader.readLine())
  6. }

La función use es equivalente al try-with-resources de Java.

Para aplicaciones de CLI, readln() es la forma estándar de obtener entrada del usuario:

  1. print("Dime tu nombre: ")
  2. val nombre = readln()
  4. print("Tu edad: ")
  5. val edad = readln().toIntOrNull() ?: 0

readln() lanza una excepción si la entrada es nula (EOF), mientras que readlnOrNull() devuelve null de forma segura.

Es la librería oficial de Kotlin para convertir objetos a formatos como JSON:

  1. @Serializable
  2. data class User(val name: String, val id: Int)
  4. // De Objeto a JSON
  5. val json = Json.encodeToString(User("Alex", 1))
  7. // De JSON a Objeto
  8. val user = Json.decodeFromString<User>(json)

Requiere el plugin de serialización en el archivo de construcción (Gradle/Maven).

Kotlin, al ejecutarse mayoritariamente sobre la JVM, delega la gestión de memoria al entorno de ejecución. Los puntos clave son:

  • Automatización: No necesitas liberar memoria manualmente (como en C++).
  • Seguridad: El sistema evita accesos a posiciones de memoria inválidas.
  • Eficiencia: La memoria se organiza en diferentes áreas según el ciclo de vida del dato.

Entender cómo se distribuye la memoria es vital para evitar problemas de rendimiento o fugas (Memory Leaks).

La memoria se divide principalmente en dos estructuras con propósitos distintos:

Característica Stack (Pila) Heap (Montículo)
Almacena Variables locales y llamadas a métodos. Objetos y variables de instancia.
Acceso LIFO (Last In, First Out). Muy rápido. Acceso aleatorio. Más lento.
Ciclo de vida Muere cuando el método termina. Vive hasta que nadie lo referencia.

Cómo interactúan el Stack y el Heap en el código:

  1. fun main() {
  2. val edad = 25 // Stack
  3. val user = User("Ana") // Ref en Stack, Objeto en Heap
  4. }

En el Stack se guarda la "dirección" (referencia), pero el cuerpo del objeto User reside en el Heap.

El Garbage Collector (GC) es un proceso automático que libera la memoria del Heap ocupada por objetos que ya no son accesibles.

  • Rastreo: El GC busca objetos que no tienen referencias activas desde el "Stack" o "Static roots".
  • Compactación: A veces mueve objetos para evitar la fragmentación de la memoria.
  • Stop-the-world: Breves pausas en la ejecución para limpiar la memoria.

No todas las referencias son iguales para el Garbage Collector:

  • Strong Reference: Es la normal (val a = User()). Mientras exista, el GC nunca borrará el objeto.
  • Weak Reference: Permite que el objeto sea recolectado si no hay referencias fuertes hacia él. Útil para caches.
  1. val refFuerte = User()
  2. val refDebil = WeakReference(refFuerte)
  4. // Si hacemos refFuerte = null, el objeto
  5. // puede ser borrado aunque exista refDebil.

Kotlin incluye herramientas pequeñas pero potentes que facilitan el desarrollo diario y la legibilidad:

  • Type Aliases: Permite dar nombres alternativos a tipos complejos o largos.
  • TODO(): Función que lanza una excepción siempre, útil para marcar código pendiente.
  • Destructuring: Descomponer objetos en variables individuales.
  1. typealias UserMap = Map<String, User>
  3. val (nombre, edad) = usuario // Destructuring
  4. TODO("Implementar lógica de guardado")

Los paquetes organizan el código en espacios de nombres. A diferencia de Java, el nombre del paquete no tiene que coincidir estrictamente con la estructura de carpetas (aunque es recomendable):

  1. package com.proyecto.app
  3. import kotlin.math.PI
  4. import kotlin.math.* // Importar todo
  6. fun main() { println(PI) }

Kotlin importa por defecto paquetes básicos como kotlin.*, kotlin.collections.* y kotlin.io.*.

Kotlin Multiplatform (KMP) permite compartir lógica de negocio entre Android, iOS, Web y Desktop:

  • Common Kotlin: Código que funciona en todas las plataformas.
  • Expect / Actual: Mecanismo para acceder a APIs nativas específicas.
  1. // En código común
  2. expect fun getPlatformName(): String
  4. // En módulo Android
  5. actual fun getPlatformName() = "Android"

Es el framework de UI basado en Jetpack Compose para crear interfaces nativas compartidas:

  • Android/iOS: UI 100% nativa con un solo código.
  • Desktop: Apps para Windows, Mac y Linux (usando Skia).
  • Web: Basado en Canvas o WebAssembly (Wasm).

Ideal para proyectos donde la lógica de validación, red y persistencia es idéntica en todos los dispositivos.
