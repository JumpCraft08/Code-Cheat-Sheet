Un programa en Java está compuesto por clases y métodos. El método main es el punto de entrada de la aplicación. Sus características principales son:

  • Sensible a mayúsculas y minúsculas.
  • Cada sentencia termina con punto y coma (;).
  • Las llaves { } definen bloques de código.
  1. public class Main {
  2. public static void main(String[] args) {
  3. System.out.println("¡Hola Mundo!");
  4. }
  5. }

Un proyecto Java básico suele tener esta estructura:

bin/ Contiene los archivos compilados (.class).
lib/ Bibliotecas externas (JAR).
Main.java Punto de entrada del programa.

Para compilar en terminal se usa javac Main.java

Para ejecutar en terminal se usa java Main

El type casting es el proceso de convertir una variable de un tipo de dato a otro. En Java, existen dos tipos principales:

    double → float → long → int

Las estructuras de control permiten alterar el flujo de ejecución de un programa.

Se dividen en tres categorías principales:

Tipo Descripción Ejemplos
Estructuras de Condición Ejecutan código según una condición if, if-else, switch
Estructuras de Bucle Repiten código mientras se cumpla una condición for, while, do-while, for-each
Estructuras de Salto Alteran el flujo de ejecución break, continue, return

Estas estructuras son fundamentales para crear programas dinámicos y eficientes, permitiendo tomar decisiones y automatizar tareas repetitivas.

La estructura if permite ejecutar un bloque de código solo si se cumple una condición.

  1. int edad = 18;
  2. if(edad >= 18) {
  3. System.out.println("Eres mayor de edad");
  4. }

if-else evalúa una condición y ejecuta una alternativa si no se cumple.

  1. int nota = 6;
  2. if(nota >= 5) {
  3. System.out.println("Aprobado");
  4. } else {
  5. System.out.println("Suspenso");
  6. }

La estructura else if permite evaluar múltiples condiciones.

  1. int nota = 8;
  2. if(nota >= 9) {
  3. System.out.println("Sobresaliente");
  4. } else if(nota >= 7) {
  5. System.out.println("Notable");
  6. } else {
  7. System.out.println("Aprobado");
  8. }

for se usa cuando sabemos cuántas veces queremos repetir algo.

  1. for(int i = 0; i < 5; i++) {
  2. System.out.println(i);
  3. }

while repite mientras la condición sea verdadera.

  1. int x = 0;
  2. while(x < 3) {
  3. System.out.println(x);
  4. x++;
  5. }

do-while ejecuta el código al menos una vez antes de comprobar la condición.

  1. int n = 1;
  2. do {
  3. System.out.println(n);
  4. n++;
  5. } while(n <= 3);

for-each se usa para recorrer elementos de un array o colección.

  1. String[] nombres = {"Ana", "Luis", "Mar"};
  2. for(String n : nombres) {
  3. System.out.println(n);
  4. }

Las estructuras de salto permiten modificar el flujo natural del programa. Son muy útiles dentro de bucles y métodos, ya que permiten:

  • Terminar un bucle antes de tiempo (break)
  • Saltar directamente a la siguiente iteración (continue)
  • Salir de un método devolviendo un valor (return)

Su uso adecuado mejora la eficiencia del código y evita condiciones complejas dentro de bucles o funciones.

break se utiliza para detener un bucle inmediatamente, independientemente de si la condición del bucle sigue siendo verdadera.

Se usa principalmente en:

  • Bucles for, while, do-while
  • Sentencias switch
  1. for(int i = 0; i < 10; i++) {
  2. if(i == 4) break;
  3. System.out.println(i);
  4. }

Salida del código: 0, 1, 2, 3

continue permite saltar el resto de instrucciones de la iteración actual y continuar con la siguiente.

Es útil para ignorar casos concretos sin detener todo el bucle.

  1. for(int i = 0; i < 5; i++) {
  2. if(i == 2) continue;
  3. System.out.println(i);
  4. }

Salida: 0, 1, 3, 4 (el número 2 es saltado)

return finaliza inmediatamente la ejecución de un método. Puede devolver un valor o nada (en métodos void).

  1. int sumar(int a, int b) {
  2. return a + b;
  3. }

Si un método contiene varias rutas posibles, return marca el final exacto de la que se ejecute.

En Java, un error es un problema que impide que el programa se compile o se ejecute correctamente. Pueden clasificarse en:

🔹 Errores de compilación

Detectados por el compilador. Son causados por:

  • Errores de sintaxis
  • Uso incorrecto de tipos de datos
  • Olvidar un punto y coma
  • Falta una llave o paréntesis
  • Variables no declaradas
  1. int x = 5
  2. // Falta el punto y coma → Error de compilación

🔹 Errores de ejecución

Ocurren mientras el programa está corriendo. Suelen estar ligados a errores lógicos o situaciones no controladas:

  • Dividir entre cero
  • Acceder fuera del rango de un array
  • Convertir tipos de forma incorrecta
  1. int a = 10 / 0;
  2. // ArithmeticException en tiempo de ejecución

Una excepción es un evento inesperado que interrumpe el flujo normal del programa. Java ofrece un sistema muy completo para manejarlas.

🔸 ¿Por qué ocurren las excepciones?

  • Acceso ilegal a memoria (arrays fuera de rango)
  • Errores aritméticos (división entre cero)
  • Entrada/Salida incorrecta
  • Conversión mal realizada entre tipos
  • Trabajar con objetos nulos (NullPointerException)

🔸 throw vs throws

throw lanza una excepción manualmente.

throws indica que un método puede lanzar una excepción.

  1. void ejemplo() throws IOException {
  2. throw new IOException("Fallo I/O");
  3. }

Las anotaciones proporcionan metadatos al compilador y al entorno de ejecución.

Anotación Descripción
@Override Indica que un método sobrescribe uno de la clase padre.
@Deprecated Marca un elemento como obsoleto.
@SuppressWarnings Oculta advertencias del compilador.
  1. @Override
  2. public String toString() {
  3.   return "Objeto";
  4. }

El bloque try…catch permite capturar excepciones para evitar que el programa termine de forma inesperada.

🔸 try

Contiene el código que puede producir un error.

🔸 catch

Contiene el código que se ejecutará si ocurre una excepción.

  1. try {
  2. int x = 5 / 0;
  3. } catch(ArithmeticException e) {
  4. System.out.println("No puedes dividir entre cero");
  5. }

Se pueden encadenar varios catch para capturar distintos tipos de excepciones.

El bloque finally se ejecuta siempre, ocurra o no ocurra una excepción. Se usa para liberar recursos como archivos, conexiones o memoria.

  1. try {
  2. int x = 10 / 0;
  3. } catch(Exception e) {
  4. System.out.println("Error detectado");
  5. } finally {
  6. System.out.println("Bloque finally ejecutado");
  7. }

Incluso si hay un return dentro del try o del catch, el finally siempre se ejecuta.

Los constructores son métodos especiales que se ejecutan cuando se crea un objeto.

  • Tienen el mismo nombre que la clase
  • No tienen tipo de retorno (ni void)
  • Inicializan los atributos del objeto
  1. public class Persona {
  2. String nombre;
  4. // Constructor
  5. public Persona(String n) {
  6. nombre = n;
  7. }
  8. }

La palabra clave this. se refiere al objeto actual y evita conflictos de nombres.

Sin this. Con this.
Puede haber confusión entre parámetros y atributos Clarifica que se refiere al atributo del objeto
  1. public class Persona {
  2. String nombre;
  3. int edad;
  5. // Constructor con this.
  6. public Persona(String nombre, int edad) {
  7. this.nombre = nombre;
  8. this.edad = edad;
  9. }
  10. }

La llamada this() permite que un constructor llame a otro constructor de la misma clase.

  • Debe ser la primera línea del constructor
  • Evita duplicación de código
  • Facilita el mantenimiento
  1. // Constructor completo
  2. public Persona(String nombre, int edad) {
  3. this.nombre = nombre;
  4. this.edad = edad;
  5. }
  7. // Constructor que llama al anterior
  8. public Persona(String nombre) {
  9. this(nombre, 0); // Edad por defecto
  10. }

Para crear objetos usamos la palabra clave new seguida del constructor de la clase.

Paso Descripción Código

2. Instanciar Crear el objeto con new new Persona("Ana", 25);
3. Asignar Guardar referencia en variable persona1 = new Persona(...);
  1. // Crear objetos usando constructor

  6. System.out.println(persona1.nombre); // "Ana"

La recursión es cuando un método se llama a sí mismo para resolver un problema.

  • Debe tener una condición base para terminar
  • Cada llamada debe acercarse a la condición base
  • Útil para problemas que se pueden dividir en subproblemas similares
  • Cuidado con el stack overflow si no hay condición base
  1. // Ejemplo: Factorial
  2. public int factorial(int n) {
  3. // Condición base
  4. if (n <= 1) {
  5. return 1;
  6. }
  7. // Llamada recursiva
  8. return n * factorial(n - 1);
  9. }
  11. // factorial(5) = 5 * 4 * 3 * 2 * 1 = 120

La abstracción permite ocultar los detalles complejos y mostrar solo las características esenciales de un objeto.

  • Se logra mediante clases abstractas e interfaces
  • Reduce la complejidad del código
  • Permite centrarse en qué hace el objeto, no en cómo lo hace
  • Facilita el mantenimiento y reutilización del código
  1. abstract class Vehiculo {
  2.   abstract void acelerar();
  3.   void encender() {
  4.     // Implementación común
  5.   }
  6. }

Una interface es un contrato que define qué métodos debe implementar una clase.

  • Todos los métodos son implícitamente públicos y abstractos
  • Las variables son implícitamente public, static y final
  • Una clase puede implementar múltiples interfaces
  • Soporta herencia múltiple de tipo
  1. interface Volador {
  2.   void volar();
  3.   void aterrizar();
  4. }
  6. class Avion implements Volador {
  7.   public void volar() { /* ... */ }
  8.   public void aterrizar() { /* ... */ }
  9. }

  6. // - Solo definir comportamiento

  • Facilita el mantenimiento del código
  • Permite validar datos antes de asignarlos
